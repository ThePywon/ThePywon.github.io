Log object

The Log object is as simple as it sounds. It is a log. You can use it to debug and it will be shown on the console.



constructor(content, color, scroll, url)


The parameter 'content' will hold the text that will be logged.
The expected value type is a String.
It is mandatory to pass in this variable.


The parameter 'color' determines the color of the border and text of the log.
The expected value type is also a string, you can put whatever working CSS color.


The parameter 'scroll' determines either or not it should scroll to the log once logged.
The expected value type is a bool.


The last parameter, 'url' determines the src attribute of the image in the log. If none is passed, there will be no image.
The expected value type is a String.





Simple example:

new Log("Hello!");


Result:




Color usage:

new Log("Hello!", "lime");


Result:




Adding an image:

new Log("Hello!", "lime", true, "../Assets/Images/GreenCheckmark.png");


Result:

Holds the content of the log. Changing it DOES NOT affect the actual content of the log.

Holds the current color of the log. Changing it DOES NOT affect the actual color of the log.

Determines if the log gets scrolled to once logged. Changing it DOES NOT affect the actual log.

Holds the link to the image of the log if there is one. Changing it DOES NOT affect the actual image of the log.

Holds the current valid state of the object. Changing it is NOT RECOMMENDED. It can get changed to false upon hitting some specific errors.

Holds the log hitself as an HTML element. Changing it is NOT RECOMMENDED. It can be used to implement some special style to it.

Holds the HTML paragraph element where the content of the log is displayed. Changing it is NOT RECOMMENDED. It can be used to implement some special style to it.

Holds the HTML image element in case the log has an image. Changing it is NOT RECOMMENDED. It can be used to implement some special style to it.





Some functions you can play with.



print()

Is called upon initialisation of the log.
Prints the log in the console.
Usage of this function is NOT RECOMMENDED.



edit(content)

Can be used to edit the log's content.
The log's content property will be updated accordingly.



setColor(color)

Can be used to edit the log's color.
Any working CSS color can be used.
The log's color property will be updated accordingly.



setImage(url)

Can be used to edit the log's image.
If there is none, it will create one.
The log's url property will be updated accordingly.



delete()

Can be used to delete the log entirely.
Doing that is irreversible and the object will be marked 'invalid' from this point on.



isValid()

Returns the '_isValid' property of the object.
Result should be a bool.



Exception object

This object's use is to debug and report errors. It can be pretty usefull in a try catch to report the error to the console.



constructor(type, content, reference = null)


The parameter 'type' will give more precisions as to what type of error this is.
The expected value type is a String.
It is mandatory to pass in this variable.


The parameter 'content' determines the content of the logged Exception.
The expected value type is also a string.
If the value is undefined, the variable 'type' will be used as the content instead.


The last parameter, 'reference' determines the object from wich it was called.
The expected value type is an Object.
This variable can make it easier for users to know where the error occured.



Simple example:

new Exception("Error!");


Result:




Type usage:

new Exception("Type", "Error!");


Result:




Adding an reference:

class someObject{
  constructor(){
    new Exception("Type", "Error!", this);
  }
}

new someObject();


Result:

Holds the type of the exception. Changing it DOES NOT affect the actual type of the exception.

Holds the referenced object from when it got logged (null if none was put in). Changing it DOES NOT affect the actual exception.

Holds the current valid state of the object. Changing it is NOT RECOMMENDED. It can get changed to false upon hitting some specific errors.

Holds the exception hitself as an HTML element. Changing it is NOT RECOMMENDED. It can be used to implement some special style to it.

InputLog object

This object's use is to get an input from the user (text, checkbox, color, file, etc...).



constructor(type, header, addition)


The parameter 'type' will be directly passed in the input tag. It determines the type of input wanted.
The expected value type is a String.
It is mandatory to pass in this variable.


The parameter 'header' determines the header of the InputLog.
The expected value type is also a string.
If the value is undefined, it will be replaced by '{type} input request'.


The last parameter, 'addition' is only used in a few occurences. When the type is a button (button, reset, submit) or a file.
In the case of a button type, it will be passed directly as the value of the button.
Otherwise, in the case of a file, it directly replaces the 'accept' parameter of the input tag.
The expected value type is a String too.



Simple example:

new InputLog("button");


Result:




Header:

new InputLog("button", "Header");


Result:




Using the addition:

new InputLog("button", "Header", "Click me!");


Result:

Holds the header of the input log. Changing it DOES NOT affect the actual header of the input log.

Holds the type of the input log. Changing it DOES NOT affect the actual type of the input log.

Holds the addition for the input log. Changing it DOES NOT affect the actual addition.

Holds the current valid state of the object. Changing it is NOT RECOMMENDED. It can get changed to false upon hitting some specific errors.

Holds the input log hitself as an HTML element. Changing it is NOT RECOMMENDED. It can be used to implement some special style to it.

Holds the input log's header as an HTML element. Changing it is NOT RECOMMENDED. It can be used to implement some special style to it.

Holds the input log's HTML input element. Changing it is NOT RECOMMENDED. It can be used to implement some special style to it.

Holds the input log's HTML submit element (only there if the input type isn't a button type). Changing it is NOT RECOMMENDED. It can be used to implement some special style to it.

Holds the input log's data. (only populated once it gets submitted). Changing it DOES NOT affect the input log.





Some functions you can play with.



print()

Is called upon initialisation of the input log.
Prints the input log in the console.
Usage of this function is NOT RECOMMENDED.



populateData()

Is called once the input log gets submitted.
Populates the input log's data.
Usage of this function is NOT RECOMMENDED.



checkValidity()

Checks if the inputed file corresponds to the passed addition (only file extensions supported)
If the type is not a file or that the addition is not defined, it simply returns true.



onSubmit(data)

Doesn't do anything on its own, but can be redefined and used that way.
It is passed the object's 'data' property right after calling populateData().



setColor(color)

Is called once the inputed value is changed if the type is color.
It sets the input log's color.



delete()

Can be used to delete the input log entirely.
Doing that is irreversible and the object will be marked 'invalid' from this point on.



isValid()

Returns the '_isValid' property of the object.
Result should be a bool.



Popup object

This object's use is to create a popup and get a response from the user.



constructor(header, callback = function(result){}, options = ["yes", "no"], img, color = "white")


The parameter 'header' determines the header of the Popup.
The expected value type is a String.
It is mandatory to pass in this variable.


The parameter 'callback' is called once the user selects a response.
The expected value type is a function.
The passed value 'result' corresponds to the user's response.


The parameter 'options' determines the response options the user will have once faced with the Popup.
The expected value type is an Array of Strings.


The parameter 'img' is passed directly in the src attribute of the Popup's image.
The expected value type is a String.
If no value is passed, there will be no image.


The parameter 'color' is passed directly in the background-color attribute of the Popup's display.
The expected value type is a String, any valid CSS color works.


Simple example:

new Popup("Header");


Result:




Using the callback:

new Popup("Header", function(result)
{
 new Log("You have selected " + result + "!");
});


Result:







Changing the options:

new Popup("Header", function(result)
{
 new Log("You have selected " + result + "!");
},
options = ["yes", "no", "maybe"]);


Result:







Adding an image:

new Popup("Header", function(result)
{
 new Log("You have selected " + result + "!");
},
options = ["yes", "no", "maybe"],
"../Assets/Images/Smiley.png");


Result:







Changing the color:

new Popup("Header", function(result)
{
 new Log("You have selected " + result + "!");
},
options = ["yes", "no", "maybe"],
"../Assets/Images/Smiley.png", "lightgrey");


Result:

Holds the popup's image url (if there even is an image in the first place). Changing it DOES NOT affect the actual image of the popup.

Holds the color of the popup. Changing it DOES NOT affect the actual color of the popup.

Holds the current valid state of the object. Changing it is NOT RECOMMENDED. It can get changed to false upon hitting some specific errors.

Holds the popup's background as an HTML element. Changing it is NOT RECOMMENDED. It can be used to implement some special style to it.

Holds the popup hitself as an HTML element. Changing it is NOT RECOMMENDED. It can be used to implement some special style to it.

Hides the popup.



Dir2D object

This object's use is to store/transform/use a direction (left, up, right, down and none).



constructor(val = "none")


The parameter 'val' determines the value of the Dir2D.
The expected value type is a String.
It is mandatory to pass in this variable.


Simple example:

var dir = new Dir2D();
new Log(dir);


Result:




Setting a value:

var dir = new Dir2D("left");
new Log(dir);


Result:
